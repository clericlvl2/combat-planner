Combat Planner — Design Prototype

Single self-contained visual source of truth (design-chain unit A). Utility / calm-neutral, dark-primary: chrome is neutral grey, color is reserved for meaning only (combatant type, health band, combat color-tag). Every core screen is shown in a mobile frame and a desktop frame, each in dark and light.

Frames: mobile 390×800 · desktop 1024×720 (design canvas, not device-accurate) Themes: dark (primary) · light Color signifiers are always paired with a label / letter / icon (PLT-5)

Contrast note (WCAG-AA)

Records every semantic foreground/background pair used in the prototype, with its measured contrast ratio in both themes, and confirms no signifier relies on color alone (each type / health band is paired with a role=img aria-label; color-tags additionally get a visible text/icon label). Threshold is WCAG-AA: ≥ 4.5:1 for the two text rows, ≥ 3:1 for the color-carrying rows (TypeStripe, health-band fill and color-tag dot are non-text UI indicators). Color rows list the worst-case (minimum) ratio across the group; every member clears the threshold. Ratios are computed against the exact background each element sits on in the markup: --type-* stripe on --surface, --health-* fill on the track --surface-3, --combat-* dot on the row --surface.

Semantic pair Token(s) Dark ratio Light ratio Non-color pairing
Primary text on background --text / --bg 16.4:1 16.1:1 n/a (neutral chrome)
Muted text on surface --text-muted / --surface 7.6:1 6.5:1 n/a (neutral chrome)
Combatant type — PC / enemy / ally --type-* ≥ 6.3:1 ≥ 3.7:1 TypeStripe (stripe count: pc=2, enemy/ally=1) + role=img aria-label naming the type (no visible text chip)
Health bands (full / wounded / bloodied / dead) --health-* ≥ 3.7:1 ≥ 3.4:1 cur/max text + role=img aria-label (name + band + cur/max); card background tint on bloodied/dead only (no visible band-label word)
Combat color-tags (8, ADR-012) --combat-* ≥ 6.3:1 ≥ 4.6:1 Dot initial letter + tag name + title text
